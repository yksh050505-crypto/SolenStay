# 앱 업데이트 절차

매니저가 새 버전을 만들고 직원들에게 배포하는 표준 절차입니다.

---

## 한눈에 보기

```
pubspec.yaml 한 줄 변경  →  APK 빌드  →  매니저 웹에서 등록  →  직원 폰 다이얼로그
```

매니저는 PC 크롬에서 매니저 계정으로 로그인하여 등록합니다. 직원은 앱을 켜기만 하면 자동으로 업데이트 다이얼로그를 봅니다.

---

## 1단계 — 버전 올리기 (pubspec.yaml)

`app/pubspec.yaml` 의 `version` 한 줄만 변경합니다.

```yaml
version: 0.2.2+5     # 예: 0.2.3+6  또는  0.3.0+6
```

### 버전 번호 규칙

| 형식 | 의미 | 예시 |
|---|---|---|
| `x.y.z` (앞) | 표시 버전 (semver, 직원에게 보임) | `0.2.3`, `0.3.0`, `1.0.0` |
| `+N` (뒤) | 빌드 번호 (정수, 비교 기준) | `+5`, `+6`, `+7` |

- **버그픽스**: `0.2.2` → `0.2.3`
- **새 기능**: `0.2.2` → `0.3.0`
- **큰 변화**: `0.2.2` → `1.0.0`
- **빌드 번호(`+N`)는 무조건 증가** — 절대 감소 금지. 연속이 아니어도 OK (예: 5 → 10 점프 가능).

> 화면에 표시되는 버전(`v0.2.2`)과 Android 시스템의 versionCode는 모두 이 한 줄에서 자동으로 따라갑니다.

## 2단계 — APK 빌드

```bash
cd C:\SolenStay\app
flutter build apk --release
```

결과: `C:\SolenStay\app\build\app\outputs\flutter-apk\app-release.apk`

처음 빌드라면 Gradle 다운로드로 10–15분, 그 다음부터는 3–5분 정도.

이름 알기 쉽게 복사:

```bash
cp build\app\outputs\flutter-apk\app-release.apk %USERPROFILE%\Downloads\SolenStay-native-0.2.3.apk
```

## 3단계 — 매니저 웹에서 등록

1. PC 크롬으로 https://solenstay-74f8e.web.app 접속
2. 매니저 PIN으로 로그인
3. **내정보 → (지원되지 않음)** 또는 매니저 대시보드를 통해 **관리자 설정** 진입
4. **앱 버전 (Android)** 섹션 → **"새 버전 등록"** 버튼
5. 다이얼로그 입력:
   - **버전** — 예: `0.2.3`
   - **버전 코드** — 예: `6` (직전 코드보다 큰 정수)
   - **APK 파일** — **"파일 선택"** 버튼 → 2단계에서 만든 APK 선택
     - Firebase Storage에 자동 업로드 + URL 자동 채움 (진행률 % 표시)
   - **변경 내용** — 예: `자동 로그인 옵션 추가, 업데이트 알림 기능 추가`
   - **필수 업데이트** — 보통 OFF. ON이면 직원이 "나중에" 버튼 없이 무조건 업데이트해야 함
6. **등록** 클릭

> 등록되는 순간 직원 폰들이 모두 그 버전 정보를 받습니다.

---

## 4단계 — 직원 경험 (자동)

직원이 앱을 켜면:
1. 홈 화면 진입 시 자동으로 새 버전 확인
2. 새 버전이 있으면 **"새 버전 업데이트"** 다이얼로그
3. 변경 내용 보고 **"지금 업데이트"** 클릭
4. Storage에서 APK 다운로드 → 알림에서 파일 눌러 설치
5. 앱이 갱신됨 (로그인 상태/데이터 유지)

---

## 실수했을 때 — 등록 취소

잘못된 버전을 등록했다면:
1. 같은 관리자 설정 → 앱 버전 (Android) → **"새 버전 등록"** 또는 우측 **"등록 취소"** 버튼
2. 다이얼로그 좌측 빨간 **"현재 등록 삭제"** 버튼
3. 확인 다이얼로그 → 삭제

삭제하면 모든 직원의 다이얼로그가 사라집니다. (이미 업데이트한 사람은 영향 없음)

> 팁: 항상 `필수 업데이트 OFF` 로 등록하면, 실수해도 직원이 "나중에" 로 빠져나올 수 있습니다.

## 사용자별 가이드

### 매니저 (관리자)
- PC 크롬에서 위 1~3단계 진행
- 폰에서는 등록 안 됨 (APK 파일 선택이 웹 전용)
- 등록 후 본인 폰의 옛 APK도 똑같이 업데이트 다이얼로그가 뜸

### 청소원·실장 (직원)
- 따로 할 일 없음
- 앱 켜면 다이얼로그 → "지금 업데이트" → 설치

### iPhone 사용자
- iOS 네이티브 앱은 **없음**
- Safari로 https://solenstay-74f8e.web.app 접속 → 홈 화면에 추가
- 새 버전은 **새로고침만 하면 자동 적용** (별도 절차 불필요)
- 업데이트 다이얼로그는 표시되지 않음 (필요 없으니까)

## 자주 묻는 질문

**Q. 버전 코드를 1에서 5로 점프해도 되나요?**
A. 됩니다. **단조 증가**만 만족하면 됨 (5 > 1). 연속일 필요 없습니다. 다만 한 번 5를 썼으면 다음은 6 이상으로만.

**Q. APK URL을 Google Drive로 해도 되나요?**
A. 됩니다. 다만 "파일 선택" 기능을 쓰면 Firebase Storage가 자동으로 처리해서 더 깔끔합니다 (토큰 URL, 일관성).

**Q. 등록한 버전을 다운그레이드(이전 버전으로 되돌리기) 할 수 있나요?**
A. 불가능합니다. 새 버전을 만들어서 올리세요. (versionCode가 단조 증가만 허용)

**Q. iPhone 사용자도 있는데 어떻게 하나요?**
A. 별도 절차 없습니다. iPhone은 웹앱(Safari)으로 접속하면 항상 최신.

**Q. `mandatory: true` 로 등록하면 어떻게 되나요?**
A. 직원 폰에서 "나중에" 버튼이 사라지고 다이얼로그를 닫을 수 없습니다. 큰 보안 패치 등 정말 강제할 때만 사용하세요.

## 개발 환경 (참고)

다른 PC에서도 빌드하려면 다음이 필요합니다:

- **Flutter SDK** (stable, 3.24+)
- **JDK 17**
- **Android SDK 36** (compileSdk)
- **Android Build Tools 35+**

빌드 토대:
- AGP 8.9.1 / Kotlin 2.1.0 / Gradle 8.11.1

이 설정은 `app/android/settings.gradle.kts`, `app/android/gradle/wrapper/gradle-wrapper.properties` 에 기록되어 있어 `git pull` 후 자동으로 따라옵니다.
